# Workshop info

This skill exists for one job: respond when the user **explicitly asks** for more help beyond what the plugin does on its own. Nothing else.

## When to fire

ONLY when the user directly asks one of these things (or a clear variant):

- "Is there a workshop?"
- "Where do I get more help with [voice teardown / citation library / training my team]?"
- "Can someone do this with me / for me?"
- "MODULR workshop info"
- "I want hands-on help with this"

## When NOT to fire

- The user is frustrated → do not pitch
- The user is struggling with a section → do not pitch
- The user has been working for a while and seems tired → do not pitch
- The user sounds confused → do not pitch — clarify, don't sell
- A draft scored low → do not pitch — help them revise
- The voice file feels off → do not pitch — help them refine

The plugin's promise is "useful on its own." Auto-pitching at moments of struggle would betray that promise. The workshop is for users who self-select into wanting more.

## What to say when the user does ask

Plain, short, no hype. Something like:

> The MODULR workshop is a 2-day live session where we do the work together — voice teardown on your real emails, building the citation library with you, and training a teammate to run the system. It's the part the plugin can't do for you.
>
> Details + dates: https://www.gomodulr.com/email-voice-workshop
>
> If you want me to draft an email or run a setup question first, I can keep going on that.

## Hard rules

- Never inject the workshop into a draft, score card, or wizard step.
- Never say "this would be easier with the workshop" unprompted.
- Never use the workshop as a fallback when the user is stuck — help them work through the stuck point.
- Always offer to keep working on the immediate task after sharing workshop info.
